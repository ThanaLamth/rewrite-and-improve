# Coincu Coin Reviews Keyword Map

Last updated: 2026-05-05

## Scope

This is a practical keyword and intent map for the `Coin Reviews` hub at `/coin-reviews/`.

It is based on:

- Google Search guidance on people-first content, descriptive titles, and crawlable internal links
- current SERP-style query patterns around coin review intent
- Coincu's current review inventory from the 9 XML exports

This is not a Semrush volume export. Use it as the structure layer first, then validate search volume, difficulty, and trend direction in Semrush.

The page should not try to rank for every coin keyword directly. Its main job is to rank and organize around `review hub` intent, then pass users and internal authority into the best child pages.

## Google-fit framing

The hub should stay aligned with Google's guidance by doing three things well:

1. Keep a clear primary purpose.
   The page is a curated research and navigation hub for coin reviews, not a catch-all archive.
2. Use descriptive link text and clear titles.
   Category labels and child links should tell users exactly what they will get.
3. Group pages by real reader intent.
   Infrastructure, meme, AI, NFT/community, and newer high-attention assets are easier for users and crawlers to understand than one flat list.

Official references:

- https://developers.google.com/search/docs/fundamentals/creating-helpful-content
- https://developers.google.com/search/docs/crawling-indexing/links-crawlable
- https://developers.google.com/search/docs/appearance/title-link

Secondary commentary to keep in mind:

- visible, descriptive anchor text still matters for internal links
- hub/category pages are stronger when they help users choose the next page instead of acting as thin archives

Commentary sources:

- https://www.seroundtable.com/google-visible-anchor-text-40925.html
- https://www.seroundtable.com/google-anchor-text-internal-links-24817.html
- https://www.seroundtable.com/location-of-internal-links-on-a-page-matter-for-seo-32838.html

### Avoid over-optimizing with

- stuffing `coin review` in every subsection title
- using near-duplicate anchor text across many links
