# Coincu Hub Page Plan

Last updated: 2026-05-05

## Goal

Turn Coincu's current archive-heavy structure into clearer topical hubs that can support:

- stronger topical authority
- cleaner internal linking
- better consolidation of review and explainer content
- less dilution from off-topic traffic plays

## Priority Order

### P1 hubs

1. Coin Reviews
2. Protocol Reviews
3. Exchange Reviews
4. Wallet Reviews
5. Ecosystem Hubs
6. Crypto Glossary

### P2 hubs

7. On-chain Movers / Why Is X Pumping
8. Airdrop / Fairdrop Guides
9. DeFi Strategy / Staking / Yield
10. Security / Scam / Incident Guides

## Recommended First Rollout

If Coincu can only build a few hubs now, do these first:

1. Upgrade `Other Reviews` into a more structured review hub
2. Create `Coin Reviews`
3. Create `Protocol Reviews`
4. Upgrade `Exchange Reviews`
5. Create `Wallet Reviews`
6. Build `TON Ecosystem Guide` as the first ecosystem child hub

## Internal Linking Rules

- Every review article should link upward to exactly one main hub.
- Every ecosystem article should link to both the ecosystem hub and any relevant coin, wallet, or protocol review.
- Every coin or protocol review should link to glossary terms where needed.
- Avoid splitting the same topic across too many near-duplicate hubs.

## Avoid

- creating hubs around celebrity net worth or trend-bait traffic
- mixing exchange, wallet, coin, and protocol intent on the same hub
- creating child ecosystem hubs before there are enough supporting articles
- treating plain archive pages as finished SEO hubs

## Suggested Build Sequence For Team

### Editorial

1. Decide which existing reviews belong to coin, protocol, exchange, or wallet buckets.
2. Write methodology intros for each review hub.
3. Fill the first cornerstone gaps before mass-producing more long-tail reviews.

### Dev / SEO

1. Build hub pages with strong intro copy and grouped links.
2. Add featured sections for cornerstone reviews.
3. Improve breadcrumbs and internal links between hub and child pages.
4. Keep archive pages indexable only if they provide real hub value.
